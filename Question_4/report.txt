3. Redirection Results:
   - `output.txt`: Captured the standard output from `echo "Hello World"`, containing the text `Hello World`.
   - `error.txt`: Captured the standard error from `ls nonexistentfile`, containing `ls: cannot access 'nonexistentfile': No such file or directory`.
   - `combined.txt`: Captured both stdout and stderr from `ls log.txt nonexistentfile > combined.txt 2>&1`, preserving the output line `log.txt` and the error message in a single file.

Explanation of Linux I/O:
Unix-like systems treat files, devices, and data streams as file descriptors. By using redirection operators such as `>` for stdout and `2>` for stderr, the shell allows separate or combined capture of output and errors into files for later inspection.
